# Fast Intro 快速介绍方案

这是一套完整的社区快速介绍方案，用于在 3-5 分钟内向现场参与者介绍周周黑客松社区。

## 📁 文件结构

```
community/
├── src/app/(public)/[locale]/fast-intro/
│   └── page.tsx                     # 页面路由
├── src/modules/public/fastintro/components/
│   ├── FastIntroHero.tsx           # 欢迎开场组件
│   ├── WhoWeAreSection.tsx         # 我们是谁 + 数据展示
│   ├── WhatWeDoSection.tsx         # 核心活动介绍
│   ├── SuccessStoriesSection.tsx   # 成功案例
│   ├── GetInvolvedSection.tsx      # 如何参与社区
│   └── BecomeMemberSection.tsx     # 成为社区成员 + CTA
└── docs/scripts/
    ├── fast-intro-detailed.md      # 详细版主持稿（逐字稿）
    └── fast-intro-outline.md       # 提纲版主持稿（关键点）
```

## 🎯 使用场景

- **目标受众**：首次或刚参加线下活动的新人
- **使用时机**：活动开始前的社区介绍环节
- **时长**：3-5 分钟
- **适用活动**：迷你黑客松、Demo Show、周末共创日等

## 🚀 快速开始

### 1. 访问网页版

在浏览器中打开：
```
https://hackathonweekly.com/fast-intro
```

或本地开发环境：
```
http://localhost:3000/fast-intro
```

### 2. 选择主持稿

根据你的经验选择合适的版本：

#### 详细版（推荐新手）
- 文件：`docs/scripts/fast-intro-detailed.md`
- 适合：第一次主持、需要完整脚本的场景
- 特点：逐字稿，包含完整话术和注意事项

#### 提纲版（推荐有经验者）
- 文件：`docs/scripts/fast-intro-outline.md`
- 适合：有经验的主持人或分享者
- 特点：核心要点 + 自由发挥空间

### 3. 现场准备

**必备材料**：
- [ ] 投影设备（电脑 + 投影仪）
- [ ] 网页访问：/fast-intro 页面
- [ ] 社群二维码（公众号/微信群）
- [ ] 活动日历链接或二维码

**可选材料**：
- [ ] 备用话筒
- [ ] 志愿者名单和位置
- [ ] 打印版主持稿（备用）

## 📋 内容结构

### 1. 欢迎开场（30秒）
- 自我介绍
- 欢迎大家
- 告知接下来讲什么

### 2. 我们是谁（45秒）
- AI 产品创造者社区
- 5+ 城市，6000+ 成员
- 200+ 活动，100+ 陪伴项目
- 爱·自由·创造

### 3. 我们做什么（1分30秒）
- 🚀 迷你黑客松 / 共学工作坊（每周一次，志愿发起）
- 🎯 Demo Show / 客厅 Demo 局
- 补充活动：周末共创日、AI 酒馆、社区不定期聚会

### 4. 成功案例（1分钟）
- ShipAny：首发破万刀
- 映壳：百万融资
- 社区官网：AI 编程一个月 0 到 1

### 5. 如何参与社区（1分钟）
- 继续参与活动（学习 AI 工具、组队打黑客松、认识伙伴）
- 贡献你的力量
  - 担任志愿者或分享者
  - 成为活动发起人（举办 AI + 感兴趣领域活动）
  - 社区支持：免费场地、宣发、SOP、志愿者招募

### 6. 成为社区成员（1分钟）
- 门槛：2 次活动 + 推荐 + 认同价值观
- 福利：知识库、代码、优先参与、项目陪伴、跨城网络

### 7. 结尾（15秒）
- Slogan："周周黑客松 - 每周末，一起创造有意思的作品！"

## 💡 使用技巧

### 主持人建议

1. **提前熟悉内容**：至少通读一遍主持稿，了解每部分重点
2. **控制时间**：整体 3-5 分钟，不要超时
3. **眼神交流**：不要只看稿子或屏幕，要看观众
4. **语气轻松**：像聊天一样，不要太正式
5. **强调关键信息**：
   - "每周末，一起创造有意思的作品"
   - "每周一次活动，由社区成员志愿发起"
   - "Demo Show / 客厅 Demo 局"

### 可选互动环节

如果时间和氛围允许，可以加入互动：

1. **第二部分后**："有多少人是第一次参加黑客松？举手！"
2. **第三部分前**："你们觉得从 0 到 1 做产品最难的是什么？"
3. **结尾前**："今天有人想好要做什么了吗？"

## 🎨 设计特点

### 视觉风格
- 继承 introppt 的紫色/蓝色渐变色系
- 大标题 + 少文字，适合现场投屏
- 关键数据可视化，3 秒内能抓住核心信息
- 单页滚动式设计，方便主持人控制节奏

### 与 introppt 的区别

| 维度 | introppt（长版） | fast-intro（快速版） |
|------|------------------|----------------------|
| **时长** | 15-20 分钟 | 3-5 分钟 |
| **目标受众** | 完全陌生的人 | 已来现场的参与者 |
| **内容深度** | 痛点、解决方案、运营模式 | 我们是谁、做什么、如何加入 |
| **视觉风格** | 信息丰富、多卡片 | 大标题、强视觉冲击 |
| **CTA** | 查看活动 | 今天之后怎么继续 |

## 🔧 技术实现

### 技术栈
- **框架**：Next.js 15 App Router
- **样式**：Tailwind CSS + shadcn/ui
- **组件**：React Server Components
- **部署**：与主站一起部署

### 本地开发

```bash
# 启动开发服务器
bun dev

# 访问页面
open http://localhost:3000/fast-intro

# 类型检查
bun type-check

# 代码格式化
bun format
```

### 修改内容

1. **修改文案**：直接编辑对应组件文件中的文本
2. **修改样式**：调整 Tailwind CSS 类名
3. **修改主持稿**：编辑 `docs/scripts/` 下的 Markdown 文件
4. **添加/删除部分**：在 `page.tsx` 中添加/删除组件引用
