## Why
目前作品提交页的「一句话介绍 / 项目链接 / 附件上传」等基础信息字段的文案与必填规则是写死在前端代码里的，导致不同活动在提交要求、提示语、必填项差异上无法通过后台配置快速适配，只能改代码发布。

## What Changes
- 为作品提交表单的基础字段（`tagline`、`demoUrl`、`attachments`）增加可配置项：`label`、`description`、`placeholder`（如适用）以及 `required`（必填/选填）与 `enabled`（是否显示）。
- 保持默认行为不变：未配置时沿用当前 UI 文案与选填/必填默认值。
- 表单校验与服务端写入规则跟随配置执行，避免仅前端校验导致绕过。

## Non-Goals
- 不做多语言文案存储/编辑（仍使用单语言字符串；多语言可作为后续独立变更）。
- 不扩展自定义字段能力（现有 `submissionFormConfig.fields` 保持不变，仅新增基础字段配置）。
- 不改变现有附件上传的存储与预览机制（仅配置是否展示与是否必填、文案）。

## Impact
- 组织者可在活动后台配置基础字段的文案与必填规则，减少改代码成本。
- 数据库无新增表（复用 `Event.submissionFormConfig` JSON 字段），但会扩展其 JSON 结构。
- 需要同时更新：活动编辑页的配置编辑器、提交/编辑作品表单 UI、服务端提交接口校验。
