## 背景

周周黑客松社区是一个使命驱动的 AI 创造者社区。商业收入用于覆盖运营成本、支付核心贡献者薪酬，盈余全部再投入社区建设，公共账目定期向成员公开。

## 许可证

本项目采用**双授权模式**：

| 内容 | 许可证 | 说明 |
|------|--------|------|
| 代码（非商用） | [AGPL-3.0](https://github.com/hackathonweekly/community/blob/main/LICENSE) | 可自由使用，但必须开源 |
| 代码（商用） | [商业授权](https://github.com/hackathonweekly/community/blob/main/LICENSE.COMMERCIAL) | 需要购买授权，联系微信 makerjackie |
| 文档/文章/图片 | [CC BY-NC-SA 4.0](https://github.com/hackathonweekly/community/blob/main/LICENSE-DOCS) | 禁止商用，允许非商业分享和修改 |

### AGPL-3.0 意味着什么？

| 你想做的事 | 是否可以 |
|-----------|---------|
| 学习、研究代码 | ✅ 可以 |
| 个人非商业项目中使用 | ✅ 可以，但需要开源 |
| 公益组织使用 | ✅ 可以，但需要开源 |
| 公司内部使用 | ❌ 需要商业授权 |
| fork 后部署商业网站 | ❌ 需要商业授权 |
| 基于代码做商业产品 | ❌ 需要商业授权 |
| 使用社区品牌标识 | ❌ 需要授权 |

### 什么算商业使用？

任何公司或组织用于业务目的的使用都需要商业授权，包括但不限于：
- 公司内部系统和工具
- 客户服务平台
- SaaS 产品
- 任何盈利项目

### 如何获得商业授权？

联系微信：**makerjackie**

我们提供灵活的授权方案，根据您的使用场景定价。

### CC BY-NC-SA 4.0 意味着什么？

项目中的非代码内容（文档、文章、图片等，位于 `apps/web/content/` 目录下）采用 CC BY-NC-SA 4.0 许可证。

| 你想做的事 | 是否可以 |
|-----------|---------|
| 阅读、学习文档内容 | ✅ 可以 |
| 非商业转载文章（注明出处） | ✅ 可以 |
| 基于文档内容做非商业二次创作 | ✅ 可以，但二次创作也必须以 CC BY-NC-SA 4.0 发布 |
| 商业媒体转载 | ❌ 不可以 |
| 使用社区品牌素材（logo 等） | ❌ 不可以，品牌素材不在此许可范围内 |

## CLA 贡献方式

我们使用 [CLA（Contributor License Agreement）](https://github.com/hackathonweekly/community/blob/main/CLA.md) 来管理贡献。

通过提交 Pull Request，您同意：
1. 您的贡献将在 AGPL-3.0 下发布
2. 您授权项目维护者将您的贡献用于商业授权
3. 您保证您的贡献是原创作品或您有权提交

```bash
git commit -m "feat: your feature description"
```

提交 PR 时，请确认勾选 CLA 同意项。

## 常见问题

**Q: 我可以 fork 这个项目用于自己的非商业社区吗？**

可以。AGPL-3.0 许可证允许你自由使用、修改和分发，但你必须开源你的修改，并且也使用 AGPL-3.0 授权。

**Q: 我可以 fork 这个项目用于商业用途吗？**

不可以。商业使用需要购买商业授权，请联系微信 makerjackie。

**Q: 我需要保留署名吗？**

是的。AGPL-3.0 要求保留版权声明和许可证文本。

**Q: 我可以使用 HackathonWeekly 的 logo 吗？**

品牌素材不在 AGPL-3.0 或 CC BY-NC-SA 4.0 许可范围内。如需使用，请联系 contact@hackathonweekly.com。

**Q: 贡献代码后，我的代码会被如何使用？**

你的代码在 AGPL-3.0 许可证下对所有人开放用于非商业用途。同时，你授权项目维护者将你的贡献用于商业授权。
