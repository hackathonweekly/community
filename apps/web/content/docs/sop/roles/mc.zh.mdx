**一句话目标**：控场节奏、带动氛围，让参与者在清晰流程中完成产出。

## 现场执行清单（活动当天）

### 活动前（T-3 ~ T-1）
- [ ] 与主理人确认主题、流程表、关键时间点与投票规则。
- [ ] 对齐组队方式与展示规则，准备开场话术与串场词。
- [ ] 与技术导师 Mentor、分享嘉宾做简短彩排，确认控时与站位。
- [ ] 确认作品提交标准与投票二维码展示方式。

### 到场准备（12:00 - 13:30）
- [ ] 与中控联调投屏/音响/计时信号。
- [ ] 与物料/场地确认投票道具、奖品与合影位置。

### 开场与组队（14:00 - 14:45）
- [ ] 5 分钟内讲清规则与时间轴，顺利完成组队。

### 开发与报时（14:45 - 17:00）
- [ ] 按节点报时，提醒提交截止与可播放确认。

### 展示与收尾（17:00 - 18:00）
- [ ] 控制展示时长与投票节奏，完成颁奖致谢与合影收尾。


## 注意事项与应急

### 关键标准
- 开场简短有力，避免“先聊 10 分钟再进入主题”。
- 超时先压缩，再跳过，保持整体节奏。
- 展示时长与投票规则必须提前讲清。

### 执行细节（按需参考）

**活动前（T-3 ~ T-1）**
- 与主理人确认活动主题、流程表与关键时间点。
- 对齐组队方式（自由/抽卡/solo）与关键词/台词限制玩法。
- 与技术导师 Mentor、分享嘉宾做 5 分钟彩排，明确控时与展示规则。
- 确认作品提交的具体标准，写进开场话术与 PPT。
- 准备串场词与过渡话术，确认场地关键信息（洗手间/WiFi/电源）。
- 预设奖项与 1 句话颁奖词，确保更多团队有上台机会。

**到场准备（12:00 - 13:30）**
- 与中控对齐计时信号与报时节点。
- 测试麦克风、音响与投屏，确认 PPT 可正常播放。
- 与物料/场地确认投票道具、奖品、合影位置。
- 确认投票二维码展示方式与倒计时道具。
- 确认路演首个“假装路演”示例内容（AI 音乐活动用往期冠军作品）与投票演示流程。

**开场与组队（14:00 - 14:45）**
- 5 分钟内讲清规则与时间轴（组队截止/作品提交/展示时长/投票方式）。
- 如开场延迟或等待，用 3-5 分钟破冰/提前学习共学文档/组队引导填空档。
- 用 PPT 明确活动流程与提交格式（视频类 MP4 优先、避免 MOV）。
- 控制灵感发布时长（1 分钟/人），及时兜底落单人员。
- 宣布组队结束，进入开发环节。

**开发与报时（14:45 - 17:00）**
- 按节点报时（30/20/10/5 分钟 + 最后 10 秒倒计时）。
- 提醒作品提交方式与截止时间，强调提交后尽快确认可播放。
- 临近结束提醒可先提交不完整版本（音频/半成品）。
- 可选：设置早提交加 1 票，鼓励提前交付。

**展示与收尾（17:00 - 18:00）**
- 路演开场先用示例作品做“假装路演”，带全场走完扫码投票全流程。
- 说明展示规则与投票方式，保持节奏流畅。
- 控制每组展示时长（建议 60-90 秒），过长作品仅放高光片段。
- 每组播完给 15-20 秒投票窗口，并邀请 1 句创作初衷分享。
- 作品过半时安排 3-5 分钟中场休息，避免审美疲劳。
- 颁奖与致谢：配乐烘托氛围，邀请志愿者上台并夸到细节。
- 合影、宣发二维码提醒、清场提示（前排蹲坐/社区旗帜入镜）。

**可选玩法：奇奇怪怪的奖**
- 准备一个“奇奇怪怪的奖”池，作为颁奖后的加码环节。
- 前 3 名必须拿奖（无论如何给他安排个奖）。
- 其他奖项随缘发放，可发可不发[旺柴]；发了要讲 1 句理由。
- 同一奖项可重复颁发，比如有几个让主持人一见钟情的[Doge]。
- 示例：一见钟情奖：看到你，还没想好理由就想颁奖。

<Callout title="大型活动可拆分" type="info">
可安排主主持 + 副主持（台下引导/收集问题/组队兜底），减轻控场压力。
</Callout>

### 常见问题与应急处理
| 问题 | 解决方案 |
| --- | --- |
| **冷场** | 主动点名互动，抛出简单问题，带头鼓掌 |
| **超时** | 眼神提示 + 近身提醒，必要时直接进入下一环节 |
| **组队混乱** | 暂停灵感发布，按技能强制配对 |
| **设备故障** | 稳住气氛，先引导短休或互动，等待技术支援 |
| **观众疲劳** | 缩短展示时长，安排中场休息 |
