**一句话目标**：让参与者快速上手，并在截止前完成可交付作品。

> 小型活动可由 1 人兼任技术导师 Mentor（教学演示 + 技术支持）；大型活动再拆分。

## 现场执行清单（活动当天）

### 活动前（T-3 ~ T-1）
- [ ] 准备共学文档与教程视频，明确导出格式与提交方式。
- [ ] 确认工具可用性与替代方案，测试提交表权限。
- [ ] 与主持人/中控对齐时间节点与展示时长。

### 13:30 - 14:00 | 早到预热
- [ ] 引导早到参与者完成注册与基础设置，播放教程视频。

### 活动当天（开场与演示）
- [ ] 15 分钟内完成教学演示，演示后迅速开始巡场。

### 巡场辅导（14:45 - 17:00）
- [ ] 定期巡场解答卡点，提示收尾与提交。

### 作品提交（16:30 - 17:00）
- [ ] 协助提交与格式核对，并确保作品可播放。

### 活动后
- [ ] 收集共性问题，更新教程与提示词。


## 注意事项与应急

### 关键标准
- 主动巡场 > 被动答疑。
- 先保证“能交付”，再追求“更完美”。
- 视频教程是保险，演示不稳定时直接切换。
- 先确认“能播放”的版本，再考虑精修。

### 执行细节（按需参考）

**活动前（T-3 ~ T-1）**
- 准备共学文档（工具链接/账号/提示词/分工建议）。
- 所有导师/助教提前完整走一遍工具链，记录常见问题与解决方案。
- 准备 3-5 分钟傻瓜式教程视频。
- 共学文档只保留最小可行步骤，避免细节过载。
- 优先选择低门槛工具，并准备替代方案。
- 共学文档标注是否需要科学上网，并提供无魔法替代路径。
- 在共学文档中写明导出格式与限制（MP4/H.264、避免 MOV、文件大小上限）。
- 录制或准备 3-8 分钟视频教程（现场翻车可切换）。
- 测试现场网络与工具可用性，准备替代方案/镜像。
- 建立作品提交表并测试权限，确认上传大小限制。
- 与主持人/中控对齐时间节点、报时方式与展示时长。
- 将 PPT/教程文件备份到本地，准备 U 盘/移动硬盘备份通道。

**13:30 - 14:00 | 早到预热**
- 引导早到参与者完成工具注册/登录与基础设置。
- 逐桌询问是否熟悉工具/是否已有主题，给出最短路径建议。
- 播放 3-5 分钟教程视频，强调“先完成再优化”。
- 引导填写主题/技能并上墙，方便后续组队。
- 提示小组先完成分工（音乐/剪辑/文案/操作）。

**活动当天（开场与演示）**
- 15 分钟内完成教学演示，核心路径尽量控制在 3-5 分钟内。
- 明确“最小可行路径”，告诉大家先做完再优化。
- 演示后 5 分钟内开始巡场。

**巡场辅导（14:45 - 17:00）**
- 每 15-20 分钟巡一圈，逐组询问卡点。
- 给出下一步动作，帮助他们收敛目标。
- 强调先出音乐/音频，视频未完成可先交。
- 每 3 组尽量配置 1 名技术助教/导师/志愿者，提升响应速度。
- 每位导师固定跟进 2-4 组（大活动指定组），确保节奏与提交。
- 按 60/30/10 分钟节点提醒收尾与提交。
- 处理账号/工具/网络问题，必要时切换替代方案。

**作品提交（16:30 - 17:00）**
- 检查分享权限，协助上传/填表。
- 核对导出格式与文件大小，必要时指导压缩或重导出。
- 确认时长 ≥30 秒，小于 3 分钟且包含音乐/音频。
- 作品提交后立即本地下载或拷贝 U 盘，确保可播放。
- 作品过长时提醒提供 60-90 秒高光片段。
- 允许先交未完成版本，后续补交高光片段或优化版。
- 核对提交数量与名单，补齐缺失信息。

<Callout title="大型活动可拆分" type="info">
可拆为教学讲师、巡场导师、提交/统计三岗。
</Callout>

### 常见问题与应急处理
| 问题 | 解决方案 |
| --- | --- |
| **工具无法访问** | 切换镜像/替代工具，或提供共享账号 |
| **队伍进度卡住** | 帮其缩小目标，先完成 MVP |
| **作品链接打不开** | 检查分享权限或重新生成链接 |
| **提交混乱** | 统一口径，设定唯一入口与截止时间 |
| **文件太大/格式不对** | 改为 MP4/H.264，降低分辨率或码率后重导出 |
| **会员限制/功能缺失** | 提供免费替代流程，或改用简化版本完成交付 |
