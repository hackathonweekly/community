**一句话目标**：把活动从筹备到收尾跑通，确保按时交付与现场体验稳定。

> 大型活动建议设置「主理人 + 执行统筹」双岗分工。

## 执行清单（筹备到收尾）

### 活动前（T-14 ~ T-1）
- [ ] 通读 [活动统筹手册](/docs/sop/manual/organizer-handbook)，确定主题/时间/场地/规模/预算。
- [ ] 产出本场活动手册（基于 [活动执行手册模板](/docs/sop/manual/activity-template)）。
- [ ] 组建核心岗位并锁定负责人（主持/中控/技术导师/物料/宣发/摄影）。
- [ ] 发布报名页与入群指引，明确参与须知与预习要求。
- [ ] 确认投票与提交方案，并对齐物料与设备清单。

### 活动当天（12:00 - 18:00）
- [ ] 提前到场检查电源/网络/投影/音响，组织志愿者短会对齐分工。
- [ ] 与主持/中控/技术导师走流程表，明确责任人与时间轴。
- [ ] 巡场协调关键节点，路演前确认作品可播放。

### 活动后（24 小时内）
- [ ] 组织合影与清场复原，完成志愿者致谢。
- [ ] 召集复盘并汇总数据，推动回顾发布与 SOP 更新。

## 注意事项与对齐

### 关键标准
- 先保障“能开始/能交付”，再优化体验。
- 无群=无流程：确保报名者全部入群。
- 统筹只抓关键节奏，不陷入细节执行。

### 执行细节（按需参考）

**活动前（T-14 ~ T-1）**
- 通读 [活动统筹手册](/docs/sop/manual/organizer-handbook)，确认标准流程与资源。
- 确定主题/时间/场地/规模/预算。
- 确认场地空调/使用时段，必要时调整时间段。
- 产出本场活动手册（基于 [活动执行手册模板](/docs/sop/manual/activity-template)）。
- 组建核心岗位并锁定负责人（主持/中控/技术导师/物料/宣发/摄影）。
- 发布报名页与入群指引，设定 T-3/T-1 提醒节奏。
- 群公告置顶交通指引/签到流程/流程表/提交与投票入口。(提前准备对应的文案，在合适的时间发送)
- 报名表增加主题/技能/队友需求（如果适用的话），提前群内接龙便于组队。
- 发布参与须知与预习要求（工具注册/提交格式/自带电脑）。
- 分享嘉宾 PPT 提前审核并安排 5-10 分钟彩排，明确控时规则。
- 确认投票与提交方案（限制、下载、备份路径）。
- 准备示例投票作品（AI 音乐活动用往期冠军作品），作为路演首个“假装路演”并演示完整投票流程。
- 确认计票方式与计票员，并提前沟通口径与统计流程。
- 作品数量过多时预设筛选/高光策略与展示上限。
- 对齐物料与设备清单（指引牌/签到表/U 盘/转接头）。
- 使用 AI 制作 PPT：先看 [AI PPT 制作指南](/docs/sop/manual/ai-ppt-guide)，模板见 [活动 PPT 模板](https://hackppt.01mvp.com/)（源码在 [GitHub](https://github.com/01mvp/sop) 的 `ai-ppt/index.html`）。

**活动当天（12:00 - 18:00）**
- 提前 2 小时到场，检查电源/网络/投影/音响。
- 开场前 1 小时召集志愿者短会：分工、注意事项、关键时间点。
- 与主持/中控/技术导师走一遍流程表，明确每个环节的责任人。
- 与主持/中控/技术导师对齐时间轴、展示时长与投票规则。
- 确认签到台、入群码、交通指引与物料到位。
- 关键节点报时与巡场协调，必要时压缩低优先级环节。
- 路演前确认作品可播放（本地下载/U 盘备份）。

**活动后（24 小时内）**
- 组织合影、清场复原、志愿者致谢。
- 召集志愿者做 10 分钟口头复盘并录音。
- 将复盘语音发送给 Jacky 或整理到 GitHub 仓库，交给 AI 更新 SOP。
- 组织参与者 & 志愿者聚餐，这是一个提升社区凝聚力的绝佳机会！
- 汇总数据与亮点，推动回顾发布。
- 收集改进点，更新 SOP 与教程。

### 常见风险与处理
| 问题 | 处理方式 |
| --- | --- |
| **关键岗位缺席** | 立即调度替补或亲自顶上 |
| **流程超时** | 砍掉低优先级环节或压缩时长 |
| **路演卡顿** | 统一下载/备份，必要时只播高光 |
| **入群遗漏** | 现场补扫 + 活动前逐一私信提醒 |
