### 📝 提交格式

每个 commit 都应包含 DCO 签名：

```bash
git commit -s -m "feat(auth): add user login feature

This commit adds OAuth authentication support with multiple providers.
The implementation includes proper error handling and security measures.

Signed-off-by: Your Name <your.email@example.com>"
```

或者使用 `git commit --amend -s` 为现有 commit 添加签名。

### ❓ 常见问题

**Q: 如果我贡献的代码包含我公司拥有的代码怎么办？**
A: 您需要确保您的公司同意将这些代码在 DCLA 和项目许可证下开源。

**Q: 我的贡献会被如何使用？**
A: 您的贡献将在 Creative Commons BY-NC 4.0 许可证下开源：
   - ✅ 个人学习、研究、教育用途
   - ✅ 开源社区的非商业协作
   - ❌ 其他用户的商业使用（禁止）
   - ✅ HackathonWeekly 的商业使用（独家授权）

**Q: 我可以撤回我的贡献吗？**
A: 一旦提交，贡献即被永久许可，无法撤回。

**Q: 如果我发现我的贡献有专利问题怎么办？**
A: 您有责任在提交前解决所有知识产权问题。

**Q: 谁拥有我贡献的代码？**
A: 您仍然是版权所有者，但您已授予他人使用的权利。
